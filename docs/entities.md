# Сущности

В сервисе сотрудников Employee есть несколько сущностей. Все они привязаны к пользователю и остаются общими для его организаций. Между пользователями сущности не передаются.

> Узнать о методах клиента для работы с сущностями можно в [_client-methods.md_](/docs/client-methods.md).

Вот на какие виды они делятся:

## `Person`

Сотрудник, его персональные данные.

| Атрибут | Значение |
|---------|----------|
| Таблица | `employee.persons` |
| Ключ | `Guid` — идентификатор сотрудника |
| Поля | `FullName` — полное имя (ФИО) <br> `InnFl` — ИНН физического лица <br> `Gender` — пол <br> `Citizenship` — гражданство <br> `BirthDate` — дата рождения <br> `BirthPlace` — место рождения <br> `Phone` — номер телефона <br> `Email` — адрес электронной почты <br> `RegistrationAddress` — адрес регистрации <br> `IdentityCard` — документ, удостоверяющий личность <br> `ForeignAddress` — адрес проживания за пределами страны |

---

## `Position`

Должность сотрудника в организации. Один сотрудник может иметь только одну должность внутри одной организации. 

| Атрибут | Значение |
|---------|----------|
| Таблица | `employee.positions` |
| Ключи | `Person` — идентификатор сотрудника <br> `Organization` — идентификатор организации |
| Поля | `Title` — название должности |

---

## `Role`

Роль сотрудника в организации. Хранит связку сотрудник–организация и его роль в этой или другой организации.

| Атрибут | Значение |
|---------|----------|
| Таблица | `employee.roles` |
| Ключи | `Organization` — идентификатор организации, в контексте которой назначена роль <br> `Type` — тип роли (доступные `RoleType`: `Head`, `ChiefAccountant`, `Sender`, `Ip`) |
| Поля | `Person` — идентификатор сотрудника, выполняющего роль <br> `PositionOrganization` — идентификатор организации, где работает сотрудник с данной ролью |
